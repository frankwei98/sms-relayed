# AGENTS.md

Instructions for coding agents working in this repository.

## Project Layout

- Root crate: Rust CLI/service for `sms-relayed`.
- Rust source: `src/`.
- Frontend app: `frontend/`, using Vite, React, TanStack Router, Tailwind CSS, Biome, and shadcn.
- Generated frontend routes live in `frontend/src/routeTree.gen.ts`.

## General Rules

- Keep changes scoped to the user request.
- Do not overwrite or revert user changes unless explicitly asked.
- Prefer existing project patterns over new abstractions.
- Use `rg` / `rg --files` for searching.
- Avoid committing build outputs, dependency folders, or local environment files.

## Rust

- Build from the repository root:

```sh
cargo build
```

- Run tests/checks from the repository root when relevant:

```sh
cargo test
cargo fmt --check
```

- Keep async/runtime code compatible with the existing Tokio/zbus structure.
- Do not log secrets, tokens, phone numbers, or full SMS bodies unless the user explicitly requests diagnostic output.

## Frontend

- Run frontend commands from `frontend/`.
- Use pnpm for package and script commands.

```sh
pnpm install
pnpm build
pnpm check
pnpm generate-routes
```

- After adding or changing TanStack Router routes, regenerate routes:

```sh
pnpm generate-routes
```

## shadcn Components

- shadcn components must be added only with this command shape:

```sh
pnpm dlx --package shadcn@latest --package zod@3.25.76 shadcn add button
```

- Do not use `pnpm dlx shadcn@latest ...`, `npx shadcn ...`, a globally installed `shadcn`, or an unpinned `zod` transitive setup.
- If a different shadcn component is needed, use the same pinned package form and replace only the final component name after `add`.
- Do not hand-copy shadcn component source unless the user explicitly asks for a manual implementation.

## Formatting

- Rust: use `cargo fmt`.
- Frontend: use Biome commands from `frontend/`.
- Keep generated files generated by their owning tools.

## Verification

- Run the narrowest relevant checks for the files changed.
- If checks cannot run because of environment constraints, report the exact command attempted and the blocker.
